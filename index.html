NIRVI – Production-Grade Backend Architecture

This diagram represents the real backend architecture I designed and implemented for the NIRVI startup. The focus was on scalability, fault isolation, asynchronous processing, and operational visibility.

  • API Gateway (APISIX) for auth, routing, and rate limiting
  • Microservices for user, vendor, payment, order, and admin domains
  • Event-driven workflows using RabbitMQ
  • Redis for hot-path caching and DB protection
  • Order assignment engine for vendor/driver matching
  • Notification engines (Firebase + AI-based content)
  • Centralized logging with Grafana & Loki

All services were containerized with Docker, with CI/CD handled via GitHub and deployments automated to VPS infrastructure.
