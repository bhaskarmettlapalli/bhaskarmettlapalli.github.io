How I Approach Backend Engineering

I believe backend engineering is about making correct trade-offs, not blindly adopting tools.
  • Designing APIs that remain stable as systems evolve
  • Choosing data stores based on access patterns
  • Using async messaging to isolate failures
  • Understanding JVM internals and concurrency models
  • Building observability from day one
I enjoy working close to the JVM — tuning memory, reducing GC pressure, and leveraging Java 21 virtual threads for high-concurrency systems.
